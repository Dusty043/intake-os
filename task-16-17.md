# TASK-0016 — Domain Foundation: Evaluation Aggregate + Agent Contracts

## Status

Planned

## Goal

Introduce the domain foundation for the full multi-agent project evaluation pipeline.

This task adds typed evaluation models, agent contracts, depth routing, validation helpers, and compatibility mappers from the new evaluation structure back to the existing legacy `IntakeAnalysisDraft`.

TASK-0016 must not change runtime behavior.

No new API routes.

No Prisma changes.

No workflow state changes.

No AI calls.

No UI changes.

This is pure application/domain foundation.

---

# Context

Current completed system:

```text
TASK-0012 — Private Server Runtime Deployment
TASK-0013 — Authenticated Internal Access & Role Resolution
TASK-0014 — Guided AI Draft Regeneration
TASK-0015 — AI Provider Router & Real Provider Adapters
```

Current AI flow:

```text
intake
→ analysis draft
→ optional guided regeneration
→ human review
→ reviewed package
→ approvals
→ distribution preview
```

TASK-0016 starts the transition from a single flat analysis draft to a full multi-agent evaluation packet.

The eventual target is:

```text
12 specialized evaluation sections
→ quality scoring
→ synthesis
→ legacy draft compatibility
→ existing human review flow remains intact
```

run agents

# Core Design Rule

The new evaluation model is additive.

The current legacy draft flow must continue to work.

The mapper introduced in this task allows:

```text
IntakeEvaluation → IntakeAnalysisDraft
```

so future tasks can generate the richer evaluation while current review/approval/distribution flows still receive the legacy draft shape.
