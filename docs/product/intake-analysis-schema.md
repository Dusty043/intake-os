# Intake Analysis Schema

## Purpose

This schema defines the structured draft generated by the AI analysis module. It is not the final approved project record.

The analysis output must be:

- schema-validated;
- versioned;
- reviewable;
- editable by humans later;
- auditable;
- blocked from direct downstream writes.

## Implemented Contract

The TASK-0005 implemented v1 contract lives in:

```text
src/application/intake-analysis.ts
```

Canonical schema version:

```text
intake-analysis-draft.v1
```

Implemented provider path:

```text
mock-intake-analysis-v1
```

The mock provider produces deterministic draft analysis only. It does not call a live model provider.

## Implemented Draft Shape

```ts
export interface IntakeAnalysisDraft {
  id: string;
  intakeId: string;
  schemaVersion: "intake-analysis-draft.v1";
  provider: "mock" | "openai" | "anthropic" | "manual";
  model: string;
  generatedAt: string;
  generatedBy: Actor;
  reviewStatus: "draft" | "accepted" | "rejected" | "superseded";
  sourceSummary: string;
  projectType: ProjectType;
  complexity: "low" | "medium" | "high" | "unknown";
  estimatedStoryPoints: number;
  confidence: number;
  recommendedTechStack: readonly string[];
  requiredEvaluationSections: readonly string[];
  infrastructureRequirements: readonly InfrastructureRequirementDraft[];
  brief: IntakeAnalysisBriefDraft;
  subtasks: readonly IntakeAnalysisSubtaskDraft[];
  assignmentRecommendation: DeveloperAssignmentRecommendationDraft;
  missingInformation: readonly string[];
  warnings: readonly string[];
}
```

## HTTP Boundary

The NestJS API source exposes the first mock endpoint:

```http
POST /intakes/:id/analysis-drafts/mock
```

Optional body:

```json
{
  "sourceInquiryText": "Client needs an internal dashboard for project throughput and developer workload.",
  "reviewerContext": "Internal-only first pass. Do not create live resources yet."
}
```

State behavior:

```text
submitted -> evaluating -> intake_review
```

Audit behavior:

```text
INTAKE_CREATED
submit
generate_evaluation
ANALYSIS_DRAFT_GENERATED
success
```

## Review Rules

- Generated analysis remains `reviewStatus = draft`.
- AI/mock output cannot approve the intake.
- AI/mock output cannot create a provisioning plan.
- Dry-run provisioning remains blocked until both human approval gates are complete.
- Human edits must preserve the original generated output as a versioned draft.
- Low-confidence analysis must require human review before approval.
- Missing blocker information should move the intake to clarification rather than approval once clarification workflow UX exists.

## Storage Notes

TASK-0005 stores analysis draft JSON on the intake snapshot and adds Prisma JSON fields:

```text
analysisDrafts
latestAnalysisDraft
```

This keeps the schema flexible during R&D. Once fields prove stable, promote high-value fields into queryable relational columns.

## Future Provider Path

The live provider path should preserve the same contract:

```text
model provider -> structured schema output -> backend validation -> draft persistence -> human review
```

No live provider should be allowed to call Monday, GitHub, email, Chat, or infrastructure APIs directly.
